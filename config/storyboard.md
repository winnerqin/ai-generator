# 生成表格

1. 表格结构：
   ◦ 表格共6列。分镜号、中文台词、说话人、情绪、描述、场景/时间 

   ​	行数 = 本分镜内“有台词的行数”（X） + 1，加的一行是表头行。

   ◦ “中文台词”、“说话人”、“情绪”三列是逐行对应的，一句台词占一行。

   ◦ “分镜号”列：每个分镜只需在第一行填写一次，格式建议为 EP[集号，两位数]-sh[分镜号，两位数]，例如 EP01-sh01。

2. “描述”列填写规范：
   ◦ 必须包含以下三个明确标出的部分，并用<br>或换行符分隔：

       1.  镜头：描述景别（如特写、中景、全景）和镜头运动（如推、拉、摇、移、跟拍）。
       2.  画面与动作：详细描述画面构图、角色（使用@引用）、道具、背景、以及角色在这一分镜内的动作、表情变化顺序。这是AI生成画面的主要依据。
       3.  音效/音乐：建议本分镜应匹配的环境音、音效和背景音乐风格。
3. “情绪”列选项：
   ◦ 必须从以下八个选项中选取其一：开心、悲伤、恐惧、愤怒、厌恶、惊讶、平静、自动。

   ◦ “自动”选项仅在以上七个标准情绪无法概括时使用。

4. “场景/时间提示”列填写规范
   ◦ 在此处直接填写该分镜整体的场景和时间提示（例如：城市公园，周日下午）。

5. 关于实现表格合并的说明：
   ◦ 由于标准Markdown (GFM) 不支持单元格合并，您在实际操作中可采用以下任一方式：

       ▪ 方式A (推荐-清晰结构)：如上文示例所示，在同一个分镜内，仅在分镜号、描述、场景/时间三列的第一行填写内容，后续行留空。这能清晰地表达合并的意图，是最通用、兼容性最好的做法。
       
       ▪ 方式B (精确控制)：在需要最终排版输出时，将Markdown文档转换为支持 rowspan 属性的HTML表格。
       
       ▪ 方式C (专用工具)：在支持扩展表格语法（如合并单元格）的Markdown编辑器（如某些笔记软件）中直接编辑。
